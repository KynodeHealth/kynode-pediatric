# v0.2.0-prepilot-local-node

Released 2026-05-06.

Pre-pilot product release for the KYNODE Pediatric Local Node. Hosted public review at [pediatric.kynode.io](https://pediatric.kynode.io/) (synthetic data only; see "Public demo mode" below).

## Added

- Offline FastAPI clinic node under `apps/local-node/`.
- SQLite local encounter storage.
- Bilingual clinic UI with Clinic mode as the default.
- Explicit synthetic walkthrough mode.
- Local assessment flow using the existing alpha packages.
- Weekly aggregate surveillance input.
- Weekly climate context input.
- Aggregate signal generation from weekly input only.
- Privacy-bounded export endpoint for zone-level JSON.
- Local audit trail with operational events only.

## Safety Boundary

- Patient-level data stays on the device.
- Weekly export is aggregate only.
- Climate notes stay local.
- Audit schema does not store PHI.
- Statistical support only. No autonomous diagnosis.

## Schema Policy

No Alembic migrations are included in this pre-pilot. If an incompatible SQLite schema is detected, the app renames the old file to `*.bak-<timestamp>` and creates a clean v0.2 database.

## Public demo mode

Setting `KYNODE_PUBLIC_DEMO_MODE=true` switches the app into a server-owned synthetic walkthrough:

- The SQLite file is reset on startup.
- All manual writes (`POST /api/assessments`, `POST /api/encounters`, `PUT /api/weekly-inputs`, `PUT /api/climate-context`, `PUT /api/node-settings`) return 403.
- Only the synthetic walkthrough flow (`POST /api/demo/load`, `POST /api/demo/assessment`, `POST /api/demo/encounter`) is callable.
- Aggregate signal and export are scoped to the synthetic zone/week/indicator triple; any other scope returns 403.
- Climate notes are stripped from `GET /api/climate-context` responses.
- A red banner is shown above every page.

This keeps `pediatric.kynode.io` shareable without ever accepting real patient data.
