# v0.1.0-pregrant-alpha

This release turns KYNODE Pediatric from a public scaffold into a working pre-pilot alpha.

The goal is narrow: show that an offline pediatric node can support point-of-care workflows and produce an aggregated climate-health signal without patient identifiers leaving the clinic.

## What Changed

- Added four standalone Python packages:
  - `growth-curves`
  - `triage-ranges`
  - `anomaly-detection`
  - `vaccinations`
- Added a bilingual static demo under `demo/`.
- Added generated synthetic demo output at `demo/data/demo-output.json`.
- Updated root documentation to describe the alpha as it exists today, not as a finished field deployment.
- Tightened clinical language around growth flags, vaccination status, anomaly signals and IMCI scope.

## Known Limits

- Demo data is synthetic.
- Growth tables are compact and support 0-60 months in this alpha.
- The Venezuela vaccination schedule is based on SVPP 2025 and needs MPPS/PAI validation before field use.
- Anomaly thresholds are transparent starting values, not field-calibrated thresholds.
- Full WHO IMCI rule coverage remains post-grant work.

## Suggested PR Summary

This PR turns the repo from a public scaffold into a pre-pilot alpha: four small offline Python packages, tests, and a bilingual demo that shows the privacy boundary.

PHI stays inside the clinic. Only an aggregated zone-level signal leaves the node.

The alpha is intentionally conservative. It does not claim field-use validation, independent clinical diagnosis, complete WHO IMCI coverage or end-to-end deployment readiness.
