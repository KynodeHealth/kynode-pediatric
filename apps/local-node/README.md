# KYNODE Pediatric Local Node

Pre-pilot offline clinic node for KYNODE Pediatric.

The Local Node captures pediatric encounters locally, runs deterministic support calculations, stores encounters in SQLite, records weekly aggregate surveillance inputs, records structured weekly climate context, and prepares an aggregate JSON export without PHI.

The public review URL runs with `KYNODE_PUBLIC_DEMO_MODE=true`. In that mode the backend owns the synthetic walkthrough state, rejects free clinical payloads, and must not be used for real patient data.

## Run Locally

```bash
python3 -m pip install -e packages/growth-curves
python3 -m pip install -e packages/triage-ranges
python3 -m pip install -e packages/anomaly-detection
python3 -m pip install -e packages/vaccinations
python3 -m pip install -e apps/local-node

python3 -m kynode_pediatric_local_node
```

Open `http://localhost:8080`.

User guide with screenshots: [../../docs/user-guide/local-node.md](../../docs/user-guide/local-node.md)

## Workflow

1. Configure node settings.
2. Enter a pediatric encounter.
3. Run local assessment.
4. Save the local encounter.
5. Enter weekly aggregate surveillance counts.
6. Enter weekly climate context.
7. Generate the aggregate signal.
8. Prepare weekly aggregate export.

The synthetic walkthrough is explicit. Clinic mode never loads synthetic data silently.

## Privacy Boundary

- Patient-level data stays on this device.
- Weekly exports contain aggregate counts only.
- Climate notes are local-only and are not exported.
- Audit events store operational metadata only.
- Statistical support only. No autonomous diagnosis.

## Public Demo Mode

Use this mode only for a public synthetic walkthrough:

```bash
KYNODE_PUBLIC_DEMO_MODE=true python3 -m kynode_pediatric_local_node
```

In public demo mode:

- manual clinical payloads are rejected;
- node settings are read-only;
- synthetic data is generated by the backend;
- climate notes are not returned by the public API;
- the demo database is reset on startup.

## Climate Context

The pre-pilot does not call a weather API and does not predict weather. Climate context is a structured weekly local observation for a zone:

- rainfall;
- flooding;
- heat alert;
- water disruption;
- vector risk;
- source;
- confidence.

Confidence meanings:

- `low`: single observation, no corroboration;
- `medium`: multiple observations or corroborated by community;
- `high`: corroborated by official authority bulletin.

No causal attribution is made between climate context and cases.

## Schema Policy

There are no Alembic migrations in the pre-pilot alpha. If the app opens an incompatible SQLite schema, it renames the old database to `*.bak-<timestamp>` and creates a clean database with the v0.2 schema.

This is intentional until the pilot storage contract is stable.
